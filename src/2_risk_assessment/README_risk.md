我来详细分析这个TCI风险评估模型中影响最终风险值的所有参数及其影响机制。

## 影响Risk值的参数体系

### 一、核心计算参数（直接影响）

根据核心公式：
```
Ã_d  = (A_d - ad_min) / (ad_max - ad_min)
R    = α·F_S - β·Ã_d
R   = 2R - 0.16
```

#### 1. **权重系数 α 和 β**
- **位置**：`config['model']['alpha']` 和 `config['model']['beta']`
- **关系**：α + β = 1（互补关系）
- **影响机制**：
  - **α 越大**：场景复杂度（F_S）权重越高 → 复杂场景下风险值越高
  - **β 越大**：驾驶能力（Ã_d）权重越高 → 能力越强风险值越低
  - **默认值**：通常 α ≈ 0.6, β ≈ 0.4（可通过权重校准优化）

#### 2. **能力归一化边界 ad_min 和 ad_max**
- **位置**：`config['model']['ad_min']` 和 `config['model']['ad_max']`
- **影响机制**：
  - **ad_min 越高**：相同 A_d 值 → Ã_d 越小 → R 越高（风险增加）
  - **ad_max 越高**：相同 A_d 值 → Ã_d 越小 → R 越高（风险增加）
  - **区间越窄**：归一化更敏感，能力差异被放大
  - **示例**：A_d=50
    - 若 [ad_min=0, ad_max=100] → Ã_d=0.5
    - 若 [ad_min=30, ad_max=70] → Ã_d=0.5（相同）
    - 若 [ad_min=0, ad_max=200] → Ã_d=0.25（风险更高）

#### 3. **线性变换系数（R → R）**
- **公式**：`R = 2R - 0.16`
- **影响**：
  - **系数 2**：放大 R 的变化范围
  - **偏移 -0.16**：整体下调风险值（使零点对应 R=0.08）

---

### 二、输入数据参数（间接影响）

#### 4. **场景复杂度 F_S**
- **来源**：
  - 空间序列：`config['paths']['fs_dir']` 下各场景CSV文件
  - 基线值：`config['model']['fs_baseline']`（label=0时使用）
- **影响机制**：
  - **F_S 越大**：场景越复杂 → R 越高
  - **空间→时间映射**：通过 `resample_1d()` 插值，片段长度影响局部F_S值
  - **示例**：
    - 高速直道：F_S ≈ 0.3 → 低风险
    - 复杂匝道：F_S ≈ 0.8 → 高风险

#### 5. **动态能力 A_d**
- **来源**：`config['paths']['ad_pkl']` 中的 `sample_dynamic_capability`
- **影响机制**：
  - **A_d 越大**：Ã_d 越大 → R 越低（β为负贡献）
  - **时序变化**：A_d 序列的波动直接影响风险曲线
  - **与 F_S 的交互**：
    - 高能力 + 简单场景 → R 可能为负（极低风险）
    - 低能力 + 复杂场景 → R 接近 +1（极高风险）

#### 6. **场景标签序列 field_labels**
- **来源**：`config['paths']['field_pkl']` 中的 `sample_field`
- **影响机制**：
  - 决定 F_S 的时空映射逻辑
  - label=0 → 使用 `fs_baseline`
  - label≠0 → 映射到对应空间序列
  - **游程编码**：连续相同标签的片段长度影响插值结果

---

### 三、阈值参数（影响风险等级划分）

#### 7. **风险等级阈值**
- **位置**：
  - `config['model']['thresh_high']`：高风险下界
  - `config['model']['thresh_low']`：中风险下界
- **影响**：
  - R ≥ thresh_high → **高风险**
  - thresh_low ≤ R < thresh_high → **中风险**
  - R < thresh_low → **低风险**
- **示例**（thresh_high=0.4, thresh_low=0.2）：
  - R = 0.5 → 高风险
  - R = 0.3 → 中风险
  - R = 0.1 → 低风险

---

### 四、参数影响矩阵总结

| 参数 | 增大时对 R 的影响 | 敏感度 | 调节建议 |
|------|-------------------|--------|----------|
| **α** | ↑（F_S权重增加） | ⭐⭐⭐⭐⭐ | 通过AUC校准优化 |
| **β** | ↓（Ã_d权重增加） | ⭐⭐⭐⭐⭐ | α的补数 |
| **ad_min** | ↑ | ⭐⭐⭐ | 基于实际能力分布 |
| **ad_max** | ↓ | ⭐⭐⭐ | 基于实际能力分布 |
| **F_S** | ↑ | ⭐⭐⭐⭐ | 取决于场景设计 |
| **A_d** | ↓ | ⭐⭐⭐⭐ | 取决于驾驶员能力 |
| **thresh_high** | 不影响R值 | - | 影响高风险比例 |
| **thresh_low** | 不影响R值 | - | 影响中风险比例 |

### 五、典型场景示例计算

假设配置：α=0.6, β=0.4, ad_min=0, ad_max=100

**场景1：新手司机遇复杂路况**
- F_S = 0.8, A_d = 30
- Ã_d = (30-0)/(100-0) = 0.3
- R = 0.6×0.8 - 0.4×0.3 = 0.48 - 0.12 = 0.36
- R = 2×0.36 - 0.16 = **0.56**（高风险）

**场景2：老司机遇简单路况**
- F_S = 0.3, A_d = 80
- Ã_d = 0.8
- R = 0.6×0.3 - 0.4×0.8 = 0.18 - 0.32 = -0.14
- R = 2×(-0.14) - 0.16 = **-0.44**（低风险）

**场景3：调整权重后的影响**（α=0.8, β=0.2）
- 场景1重算：R = 0.8×0.8 - 0.2×0.3 = 0.58，R = **1.0**（更高风险）
- 说明：提高 α 使模型对场景复杂度更敏感

### 六、参数优化建议

1. **权重校准**：使用 `compute_weight_calibration()` 通过网格搜索找到最优 α（最大化AUC）
2. **能力边界**：根据实际数据分布设置 ad_min/ad_max（如用5%和95%分位数）
3. **阈值标定**：根据历史事故数据调整 thresh_high/thresh_low（平衡召回率和精确率）
4. **场景F_S验证**：确保各场景F_S值符合实际复杂度排序

这个模型的优势在于物理意义明确：**风险 = 场景压力 - 驾驶员应对能力**。
