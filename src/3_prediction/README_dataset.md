# MT-JP 联合预测模型 — 数据集输入说明

### 数据来源（3个核心文件）

| 文件 | 路径配置键 | 核心内容 |
|------|------------|----------|
| 原始多模态数据 | `paths.raw_pkl` | 原始多模态时序信号：<br>- act：行为信号<br>- eye：眼动信号<br>- phy：生理信号 |
| 能力评估结果 | `paths.cap_pkl` | 能力评估结果：<br>- `sample_fluctuations`：各样本逐窗口能力波动量 $A_{fl}$<br>- `sample_field`：各样本逐窗口场景标签<br>- `sample_window_counts`：各样本窗口数 |
| 风险评估结果 | `paths.risk_csv` | 风险评估结果（每窗口）：<br>- `R`：风险度<br>- `risk_level`：风险等级（低/中/高）<br>- `F_S`：环境特征<br>- `field_label`：场景标签 |

---

### 原始信号结构（适配 `raw_data_17.pkl`）

| 模态 | 键名 | 采样率 | 维度 | 字段含义 |
|------|------|--------|------|---------|
| 行为 | `act` | 由配置 `window.act_hz` 指定 | 9 维 | [0-7] 行为特征（方向盘转角、角速度、油门、刹车、车速、纵向加速度、横向加速度、相对理想路径偏移量）<br>[8] 路段类型（场景标签，众数投票） |
| 眼动 | `eye` | 由配置 `window.eye_hz` 指定 | 5 维 | [0-4] 眼动特征（gaze_x、gaze_y、眨眼频率等，具体依原始数据而定） |
| 生理 | `phy` | 由配置 `window.phy_hz` 指定 | 4 维 | [0-3] 生理特征（心率均值、HRV、BVP、呼吸等，具体依原始数据而定） |

每个键均为 **List，长度 = 驾驶人样本数**，每个元素为该驾驶人的时序 `ndarray`。

---

### 模型输入窗口设计（论文 §5.1）
模型采用「历史序列 → 未来单步」的预测范式：
- **历史长度**：$T$ 步（由配置 `model.seq_len` 指定，每步对应 3s 能力评估窗口，共 $T \times 3$s 历史序列）
- **预测目标**：第 $T+1$ 步（未来 3s 窗口）的能力值/风险度/风险等级
- **特征维度**：$D = 18$ 维（行为 8 维 + 眼动 5 维 + 生理 4 维 + 环境 1 维）

#### 模型输入张量 X
```
X : np.ndarray  shape = (N, T, D=18)   dtype=float32
```
**D=18 维特征详细组成（与 `feature_names` 一一对应）：**
| 索引 | 特征名 | 模态 | 说明 |
|------|--------|------|------|
| 0 | `act_steer_angle` | 行为 | 方向盘转角（取绝对值） |
| 1 | `act_steer_vel` | 行为 | 方向盘角速度（取绝对值） |
| 2 | `act_throttle` | 行为 | 加速踏板（取绝对值） |
| 3 | `act_brake` | 行为 | 制动踏板（取绝对值） |
| 4 | `act_speed` | 行为 | 车速（保留原值） |
| 5 | `act_lon_acc` | 行为 | 纵向加速度（取绝对值） |
| 6 | `act_lat_acc` | 行为 | 横向加速度（取绝对值） |
| 7 | `act_lat_off` | 行为 | 相对理想路径偏移量（取绝对值） |
| 8 | `eye_gaze_x` | 眼动 | 眼动 x 方向（窗口均值） |
| 9 | `eye_gaze_y` | 眼动 | 眼动 y 方向（窗口均值） |
| 10 | `eye_blink_freq` | 眼动 | 眨眼频率（窗口均值） |
| 11 | `eye_blink_freq_rep` | 眼动 | 眨眼频率冗余（窗口均值，依原始数据而定） |
| 12 | `eye_dispersion` | 眼动 | 眼动分散度（窗口均值，依原始数据而定） |
| 13 | `phy_hr_mean` | 生理 | 心率均值（窗口均值） |
| 14 | `phy_hrv` | 生理 | 心率变异性（取绝对值） |
| 15 | `phy_bvp` | 生理 | 血容量脉搏（窗口均值） |
| 16 | `phy_resp` | 生理 | 呼吸（窗口均值） |
| 17 | `F_S` | 环境 | 环境特征（来自风险评估 CSV，已归一化至 [0,1]，**不参与 Z-score 标准化**） |

---

### 预测标签（均对应未来第 $T+1$ 步）

| 字段 | 类型 | 取值范围 | 含义 |
|------|------|----------|------|
| `y_ability` | float32 | [0, 1] | 未来步归一化能力值 $\tilde{A}_d$（由 $A_{fl}$ 线性映射至 [0,1]） |
| `y_risk_reg` | float32 | 依实际数据而定 | 未来步风险度 $R$ |
| `y_risk_cls` | int64 | {0, 1, 2} | 未来步风险等级（0=低/1=中/2=高，由配置 `risk_level_map` 映射） |

---

### 数据集划分
按**驾驶人**（`sample_idx`）分层划分，保证各驾驶人平均能力的高/中/低三组在训练/验证/测试集中比例均衡：
- 划分比例由配置 `split.train_ratio` / `split.val_ratio` 指定（如 70%:10%:20%）。

---

### 输出文件结构
数据集最终保存路径由配置 `paths.output_pkl` / `paths.output_stats` 指定，文件名附带**数据增强关键参数**：

#### 1. 主数据集文件：`dataset_xxx.pkl`
文件内为字典结构，包含 `train / val / test` 三个 split，每个 split 对应以下内容：
```python
{
  "X"         : np.ndarray  (N, T, D=18)  float32  # 历史输入序列
  "y_ability" : np.ndarray  (N,)            float32  # 未来3s能力标签
  "y_risk_reg": np.ndarray  (N,)            float32  # 未来3s风险度回归标签
  "y_risk_cls": np.ndarray  (N,)            int64    # 未来3s风险分类标签
  "meta"      : pd.DataFrame (N, 3 or 4)    # 元信息（可追溯性）
                - `sample_idx`：驾驶人索引
                - `window_idx`：目标步（未来3s）的窗口索引
                - `field_label`：目标步的场景标签
                - `augmented`（可选）：是否为数据增强样本（仅启用增强时存在，True/False）
}
```
此外，文件根层级还包含：
- `norm`：字典，含 `mu`（训练集特征均值，shape=(18,)）、`sigma`（训练集特征标准差，shape=(18,)），供推理时 Z-score 标准化使用；**注意：第 17 维（F_S）的 `mu=0`、`sigma=1`，不参与标准化**。
- `feature_names`：列表，对应 18 维特征的名称（与输入张量维度一一对应）。

#### 2. 特征统计文件：`dataset_stats_xxx.csv`
CSV 格式，记录每个特征的训练集统计量（供推理/部署时标准化）：
| 列名 | 内容 |
|------|------|
| `feature` | 18 维特征的名称（如 `act_steer_angle`、`F_S` 等） |
| `train_mean` | 训练集该特征的均值（`F_S` 均值固定为 0） |
| `train_std` | 训练集该特征的标准差（`F_S` 标准差固定为 1） |

#### 文件名规则
文件名后缀 `xxx` 为数据增强关键参数，由配置 `augmentation` 动态生成，示例：
- 禁用增强：`dataset_aug-False.pkl`
- 启用增强（仅训练集、高斯噪声 std=0.05）：`dataset_aug-True_onlyTrain-True_gaussStd-0.05.pkl`
- 启用增强（含时间扭曲、特征丢弃等）：`dataset_aug-True_onlyTrain-True_gaussStd-0.05_timeWarpSigma-0.2_featDrop-0.1.pkl`

---

### 补充说明
1. **特征后处理**：
   - **行为（Act）**：除车速（`act_speed`）外，其余 7 维（0-3, 5-7）均取绝对值；
   - **眼动（Eye）**：仅做窗口均值（原始数据若已含绝对值处理则保持）；
   - **生理（Phy）**：HRV（`phy_hrv`）取绝对值。
2. **数据增强**：仅作用于训练集，且在 Z-score 标准化前执行；增强样本的 `meta` 会复制原始样本信息并标记 `augmented=True`。
3. **Z-score 标准化**：统计量（`mu`/`sigma`）仅从训练集计算；`F_S`（第 17 维）不参与标准化。
4. **中间文件**：脚本会自动生成逐窗口对齐表 `mtjp_window_aligned.csv`，保存至输出目录同级，便于调试。

---

### 总结
1. 数据集核心逻辑是「$T \times 3$s 历史多模态序列 → 预测未来 3s 能力/风险」，输入维度固定为 `(N, T, 18)`；
2. 输出文件包含完整的训练/验证/测试集及标准化统计量，文件名附带增强参数便于版本管理；
3. `meta` 元信息保留样本溯源能力，支持后续分析不同驾驶人/场景的预测效果。
